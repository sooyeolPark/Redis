# Redis List 타입

Redis List는 **순서가 있는 문자열 목록**이다. 연결 리스트 구조로 구현되어 있어 양 끝(Head/Tail)에서의 삽입·삭제가 O(1)로 빠르며, 큐·스택·최근 이력 등 다양한 자료구조로 활용할 수 있다.

---

## List의 주요 특징

### 1. 연결 리스트(Linked List) 구조

Redis List는 내부적으로 **연결 리스트**로 구현되어 있다.

- **Head와 Tail 빠른 접근:** 리스트의 **맨 앞(Head)**과 **맨 뒤(Tail)**에 대한 접근이 빠르다. 양 끝에서의 추가·제거가 효율적으로 동작한다.
- **중간 삽입/삭제:** 인덱스로 중간 위치에 접근할 때는 Head 또는 Tail에서 해당 위치까지 순회해야 하므로, 인덱스가 클수록 비용이 늘어난다. 따라서 List는 “양 끝” 연산 위주로 사용하는 것이 유리하다.

### 2. 양방향 삽입/삭제 및 O(1) 시간 복잡도

List의 **양 끝**에서 삽입·삭제를 수행하는 명령은 모두 **O(1)**이다. 리스트 길이와 관계없이 일정한 시간 안에 처리된다.

| 구분 | 명령어 | 동작 |
|------|--------|------|
| **삽입** | **LPUSH** | 리스트 **왼쪽(Head)**에 요소 추가 |
| **삽입** | **RPUSH** | 리스트 **오른쪽(Tail)**에 요소 추가 |
| **삭제** | **LPOP** | 리스트 **왼쪽(Head)**에서 요소 제거 후 반환 |
| **삭제** | **RPOP** | 리스트 **오른쪽(Tail)**에서 요소 제거 후 반환 |

여러 값을 한 번에 넣을 수 있다. 예: `LPUSH key a b c` → 리스트는 `[c, b, a]` 순서가 된다.

### 3. 인덱스 접근 가능

List는 **0부터 시작하는 인덱스**로 특정 위치의 요소를 조회할 수 있다.

- **LINDEX:** `LINDEX key index`로 해당 **인덱스**에 있는 요소 하나를 반환한다. 인덱스는 Head가 0, Tail 방향으로 1, 2, … 이며, **음수 인덱스**도 지원한다. -1은 마지막, -2는 뒤에서 두 번째를 의미한다.
- **주의:** LINDEX는 연결 리스트이므로, 인덱스까지 **순차 순회**가 필요하다. 인덱스가 클수록 시간이 길어지며, 시간 복잡도는 **O(N)**이다. 자주 쓰는 위치는 가능하면 Head나 Tail 쪽으로 두는 것이 좋다.

### 4. 최대 저장 가능 요소 수

한 List에 저장할 수 있는 요소 개수는 **최대 2^32 - 1개**(약 42억 개)이다. 실무에서는 메모리와 성능을 고려해 필요한 범위 내에서 사용하는 것이 좋다.

---

## LPUSH / RPUSH (삽입)

List의 **Head** 또는 **Tail**에 요소를 추가하는 명령어이다.

### LPUSH (List Head에 추가)

- **동작:** `LPUSH key value [value ...]`는 지정한 키의 List **맨 앞(Head)**에 값을 넣는다. **왼쪽에서 삽입**하는 것과 같다.
- **여러 값:** 여러 값을 주면 명령에 적힌 **순서대로** Head 쪽에 쌓인다. 예: `LPUSH mylist a b c` → 리스트는 `c, b, a` 순서가 된다.
- **시간 복잡도:** **O(1)** (한 개 추가 시). 여러 개 추가 시에는 추가하는 개수에 비례한다.

### RPUSH (List Tail에 추가)

- **동작:** `RPUSH key value [value ...]`는 지정한 키의 List **맨 뒤(Tail)**에 값을 넣는다. **오른쪽에서 삽입**하는 것과 같다.
- **여러 값:** 여러 값을 주면 명령에 적힌 **순서대로** Tail 쪽에 붙는다. 예: `RPUSH mylist a b c` → 리스트는 `a, b, c` 순서가 된다.
- **시간 복잡도:** **O(1)** (한 개 추가 시).

키가 없으면 새 List가 만들어진다. 해당 키가 List 타입이 아니면 오류가 발생한다.

---

## LPOP / RPOP (삭제 및 반환)

List의 **Head** 또는 **Tail**에서 요소를 **하나 제거**하고, 그 **값을 반환**하는 명령어이다.

### LPOP (List Head에서 제거)

- **동작:** `LPOP key`는 List의 **맨 왼쪽(Head)** 요소를 제거하고, 그 값을 반환한다. **왼쪽에서 꺼내기**에 해당한다.
- **리스트가 비어 있으면:** nil을 반환한다.
- **시간 복잡도:** **O(1)**.

### RPOP (List Tail에서 제거)

- **동작:** `RPOP key`는 List의 **맨 오른쪽(Tail)** 요소를 제거하고, 그 값을 반환한다. **오른쪽에서 꺼내기**에 해당한다.
- **리스트가 비어 있으면:** nil을 반환한다.
- **시간 복잡도:** **O(1)**.

Redis 6.2 이상에서는 `LPOP key count`, `RPOP key count`처럼 **여러 개를 한 번에** 제거하고 배열로 반환하는 형태도 지원한다.

---

## LRANGE (범위 조회)

지정한 **인덱스 구간**의 요소들을 한 번에 조회하는 명령어이다.

- **문법:** `LRANGE key start stop`. `key`는 List의 키, `start`와 `stop`은 **포함**되는 구간의 시작·끝 인덱스이다.
- **지정 범위의 요소 조회:** start 이상 stop 이하 인덱스에 있는 요소들을 **순서대로** 반환한다.

### 0부터 시작하는 인덱스

- 인덱스는 **0부터** 시작한다. Head가 0, 그다음이 1, 2, … 순서이다.
- **예:** `LRANGE mylist 0 2` → **처음 3개** 요소(인덱스 0, 1, 2)를 반환한다.

### 음수 인덱스 지원

- **-1**은 **마지막** 요소, **-2**는 뒤에서 두 번째, **-3**은 뒤에서 세 번째를 의미한다.
- **예:** `LRANGE mylist -3 -1` → **마지막 3개** 요소를 반환한다.
- **전체 조회:** `LRANGE mylist 0 -1`로 리스트 전체를 가져올 수 있다.

**시간 복잡도:** O(S+N). S는 start의 절댓값, N은 구간 길이이다. Head나 Tail 근처를 조회할수록 빠르다.

---

## LLEN / LINDEX (길이·단일 요소 조회)

### LLEN (길이 조회)

- **동작:** `LLEN key`는 해당 List에 들어 있는 **요소 개수**를 반환한다.
- **시간 복잡도:** **O(1)**. Redis가 길이를 내부적으로 유지하므로 **즉시 반환**된다.
- **키가 없거나 빈 List:** 0을 반환한다.

### LINDEX (특정 인덱스 요소 조회)

- **동작:** `LINDEX key index`는 해당 List에서 **지정한 인덱스**의 요소 **한 개**를 반환한다.
- **시간 복잡도:** **O(N)**. 연결 리스트이므로 Head(또는 Tail)에서 해당 인덱스까지 **순회**해야 한다. 인덱스가 클수록 느려진다.
- **인덱스 범위:** 0은 첫 요소, -1은 마지막 요소이다. 범위를 벗어나면 nil을 반환한다.

## List 활용: Queue와 Stack

LPUSH, RPUSH, LPOP, RPOP 조합으로 **큐(Queue)**와 **스택(Stack)**을 만들 수 있다.

### Queue (큐) — FIFO (First-In-First-Out)

먼저 넣은 요소를 먼저 꺼내는 구조이다.

- **RPUSH + LPOP:**  
  **Tail에 넣고(RPUSH)** **Head에서 꺼낸다(LPOP)**. 넣은 순서대로 꺼내진다.
- **LPUSH + RPOP:**  
  **Head에 넣고(LPUSH)** **Tail에서 꺼낸다(RPOP)**. 역방향이지만 역시 FIFO이다.

**활용:** 작업 큐, 메시지 큐, **Producer-Consumer** 패턴. Producer가 RPUSH로 작업을 넣고, Consumer가 LPOP으로 하나씩 꺼내 처리하는 방식에 적합하다.

### Stack (스택) — LIFO (Last-In-First-Out)

나중에 넣은 요소를 먼저 꺼내는 구조이다.

- **LPUSH + LPOP:**  
  **Head에 넣고(LPUSH)** **Head에서 꺼낸다(LPOP)**. 가장 최근에 넣은 것이 먼저 나온다.
- **RPUSH + RPOP:**  
  **Tail에 넣고(RPUSH)** **Tail에서 꺼낸다(RPOP)**. 동일하게 LIFO이다.

**활용:** **최근 이력 관리**, 실행 취소(Undo) 목록, 방문 기록 등 “가장 최근 N개”를 다룰 때 유용하다.

---

## Blocking POP (BLPOP / BRPOP)

일반 LPOP/RPOP은 List가 비어 있으면 즉시 nil을 반환한다. **BLPOP**, **BRPOP**은 List에 요소가 **들어올 때까지** 또는 **지정한 시간까지** 대기한 뒤, 있으면 꺼내서 반환한다.

- **BLPOP key [key ...] timeout:** 주어진 키들 중 **아무 List에서나** 먼저 요소가 생기면 그 List의 **Head**에서 하나 제거하고 (키, 값) 쌍을 반환한다. timeout 초 동안 아무 것도 없으면 nil을 반환한다.
- **BRPOP:** 동일하되 **Tail**에서 제거한다.

**활용:** Consumer가 “데이터가 올 때까지 블로킹”하는 **Producer-Consumer** 큐, 간단한 메시지 큐 구현에 사용된다. Redis 6.2 이상에서는 **BLMOVE** 등으로 한 명령으로 원자적 이동도 가능하다.
