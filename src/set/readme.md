# Redis Set 타입

Redis Set은 **중복 없는 문자열의 집합**이다. 순서가 없고, 멤버 존재 여부 확인이 O(1)로 빠르며, 교집합·합집합·차집합 등 집합 연산을 지원한다. 태그, 좋아요 사용자 목록, 중복 제거 등에 적합하다.

---

## Set의 주요 특징

### 1. 중복 없는 컬렉션 (Unique Collection)

Set에는 **동일한 값이 두 번 들어가지 않는다**.

- **유일한 값만 저장:** 같은 멤버를 여러 번 추가해도 내부에는 하나만 유지된다. 중복 제거가 자동으로 이루어진다.
- **활용:** 태그 목록, 좋아요/찜한 사용자 ID, 접근 가능한 권한 목록 등 “이 값이 있는지”만 중요할 때 사용하기 좋다.

### 2. 순서 없음 (Unordered)

Set은 **저장 순서를 보장하지 않는다**.

- **삽입 순서 보장 안 됨:** SADD로 넣은 순서와 SMEMBERS 등으로 꺼낸 순서가 일치하지 않을 수 있다. 인덱스나 “첫 번째/마지막” 개념이 없다.
- **정렬이 필요하면:** Sorted Set을 사용하거나, 애플리케이션에서 정렬해야 한다.

### 3. O(1) 멤버십 체크 (Membership Check)

**특정 값이 Set에 있는지** 확인하는 연산이 **O(1)**이다.

- **매우 빠른 검색:** 내부적으로 해시 테이블 등을 사용하므로, 원소 개수와 관계없이 일정한 시간 안에 “포함 여부”를 판단할 수 있다.
- **활용:** 블랙리스트/화이트리스트, 캐시 키 집합, 대량 데이터에서의 중복·존재 여부 확인 등에 유리하다.

### 4. 집합 연산 지원 (Set Operations)

여러 Set에 대해 **교집합·합집합·차집합**을 한 번에 계산할 수 있다.

- **SINTER (교집합):** 모든 Set에 **공통으로** 들어 있는 요소만 반환한다.
- **SUNION (합집합):** 모든 Set의 요소를 **합치고 중복을 제거**한 결과를 반환한다.
- **SDIFF (차집합):** **첫 번째 Set**에는 있지만, 그 뒤에 나열한 Set들에는 없는 요소만 반환한다.

태그 기반 검색, 권한/그룹 비교, 추천 후보 계산 등에 활용할 수 있다.

## SADD / SREM (요소 추가·제거)

### SADD (요소 추가)

- **동작:** `SADD key member [member ...]`는 지정한 Set에 **멤버를 추가**한다. 이미 있는 멤버는 다시 넣어도 **무시**된다.
- **중복 무시, 유일성 보장:** Set의 특성상 같은 값은 한 번만 저장되므로, SADD는 “아직 없던 값만” 추가한다.
- **반환값:** 이번 명령으로 **실제로 추가된 멤버 개수**를 반환한다. 이미 있던 멤버는 제외된다. 예: 이미 `apple`이 있는 Set에 `SADD fruits apple banana`를 하면 1만 반환된다.
- **시간 복잡도:** 한 개 추가 시 **O(1)**, 여러 개 추가 시 추가하는 개수에 비례한다.

### SREM (요소 제거)

- **동작:** `SREM key member [member ...]`는 지정한 Set에서 **멤버를 제거**한다. Set에 없는 멤버를 지정하면 **무시**된다.
- **존재하는 요소만 제거:** 제거 대상 중 Set에 실제로 있는 것만 삭제되며, 없던 값은 에러 없이 넘어간다.
- **반환값:** 이번 명령으로 **실제로 제거된 멤버 개수**를 반환한다. 예: Set에 `apple`만 있고 `SREM fruits apple grape`를 하면 1만 반환된다.
- **시간 복잡도:** 한 개 제거 시 **O(1)**.

키가 없으면 SADD 시 새 Set이 생성되고, SREM 시 0이 반환된다. 해당 키가 Set 타입이 아니면 오류가 발생한다.

## Set 조회: SMEMBERS / SISMEMBER

### SMEMBERS (모든 요소 조회)

- **동작:** `SMEMBERS key`는 해당 Set에 있는 **모든 멤버**를 반환한다.
- **반환값:** 멤버들이 담긴 **배열**이다. 순서는 보장되지 않는다.
- **시간 복잡도:** **O(N)**. N은 Set의 크기이므로, 원소가 많을수록 부담이 커진다. “전체 목록”이 꼭 필요할 때만 사용하고, 존재 여부만 확인할 때는 SISMEMBER를 쓰는 것이 좋다.

### SISMEMBER (멤버 존재 확인)

- **동작:** `SISMEMBER key member`는 해당 Set에 **지정한 멤버가 있는지** 확인한다.
- **반환값:** 있으면 **1**, 없으면 **0**을 반환한다. 불리언처럼 사용할 수 있다.
- **시간 복잡도:** **O(1)**. Set 크기와 관계없이 빠르다. “이 사용자가 좋아요를 눌렀는지”, “이 태그가 있는지” 등 단일 멤버 확인에 적합하다.

## 집합 연산 (SINTER, SUNION, SDIFF)

두 개 이상의 Set을 넘겨 **교집합·합집합·차집합**을 계산한다. 인자 순서와 “첫 번째 Set”의 의미만 주의하면 된다.

### SINTER (교집합)

- **동작:** `SINTER key [key ...]`는 **모든 인자 Set에 공통으로 포함된** 멤버만 반환한다.
- **의미:** 논리적으로 **AND**에 해당한다. “A이면서 B인 것”을 구할 때 쓴다.
- **예:** `SINTER tag:java tag:backend` → Java이면서 backend 태그가 붙은 항목 ID 집합.
- **시간 복잡도:** O(N × M). N은 가장 작은 Set 크기, M은 Set 개수이다.

### SUNION (합집합)

- **동작:** `SUNION key [key ...]`는 **인자로 준 모든 Set의 멤버를 합친 뒤, 중복을 제거한** 결과를 반환한다.
- **의미:** 논리적으로 **OR**에 해당한다. “A이거나 B인 것”을 구할 때 쓴다.
- **예:** 여러 태그 Set을 SUNION하면 “이 중 하나라도 붙은” 항목들을 모을 수 있다.
- **시간 복잡도:** 모든 Set의 원소 수 합에 비례한다.

### SDIFF (차집합)

- **동작:** `SDIFF key [key ...]`는 **첫 번째 Set**에는 있지만, **두 번째 이후 Set 어디에도 없는** 멤버만 반환한다.
- **의미:** 논리적으로 **첫 Set에서 나머지 Set들을 뺀 것(MINUS)**이다. “A에는 있지만 B에는 없는 것”을 구할 때 쓴다.
- **예:** `SDIFF all_users banned_users` → 차단되지 않은 사용자 ID 집합.
- **시간 복잡도:** 모든 Set의 원소 수 합에 비례한다.

SINTERSTORE, SUNIONSTORE, SDIFFSTORE는 연산 결과를 **새 Set에 저장**하는 명령이다. 결과를 여러 번 쓰거나 다른 연산과 조합할 때 유용하다.

## 기타 유용한 명령

### SCARD (크기 조회)

- `SCARD key`는 Set에 들어 있는 **멤버 개수**를 반환한다. **O(1)**.
- 예: 게시글별 좋아요 수(`SCARD post:1:likes`).

### SPOP (무작위 제거)

- `SPOP key [count]`는 Set에서 **무작위로** 멤버를 **최대 count개** 꺼내서 반환하고, 그 멤버들은 Set에서 제거된다. count 생략 시 1개.
- **O(1)** ~ O(N). 랜덤 추첨, 쿠폰 지급 등에 쓸 수 있다.

### SRANDMEMBER (무작위 조회, 제거 없음)

- `SRANDMEMBER key [count]`는 Set에서 **무작위로** 멤버를 **최대 count개** 반환하지만, Set 내용은 **변경하지 않는다**. count가 음수면 같은 원소가 여러 번 나올 수 있다.
- “미리보기”, “랜덤 추천” 등에 사용할 수 있다.
